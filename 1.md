# CollectAI ? What Was Built

## Overview

Starting from a backend-only project with a blank frontend, the following was completed in one session: Phase 1 frontend UI, Phase 3 AI integration, bug fixes, and full Docker deployment setup.

---

## 1. Bug Fixes

### Critical: `deleteInvoice` was deleting random documents
**File:** `server/controllers/invoice.controller.js`
- **Before:** `await Invoice.deleteOne()` ? no filter, deleted any random invoice
- **After:** `await invoice.deleteOne()` ? instance method on the found document

### Security: `updateInvoice` accepted raw `req.body`
- **Before:** `findByIdAndUpdate(id, req.body)` ? client could overwrite `user`, `_id`, etc.
- **After:** Destructures only safe fields: `clientName`, `clientEmail`, `amount`, `dueDate`, `description`, `status`

### CORS was open to all origins
- **Before:** `app.use(cors())` ? allows any domain
- **After:** `app.use(cors({ origin: process.env.CLIENT_URL }))` ? restricted to frontend URL

## 3. Frontend Built from Scratch (Phase 1 UI)

The entire React frontend was built ? `client/src/App.jsx` was previously just "Hello world!".

### Infrastructure
| File | Purpose |
|------|---------|
| `client/src/services/api.js` | Axios instance with `/api` base URL + auto Bearer token header |
| `client/src/context/AuthContext.jsx` | Auth state: login, register, logout, user, loading |

### Components
| Component | Purpose |
|-----------|---------|
| `Layout.jsx` | Fixed sidebar (dark slate) + main content area |
| `ProtectedRoute.jsx` | Redirects unauthenticated users to `/login` |
| `StatusBadge.jsx` | Coloured pill: pending=amber, paid=green, overdue=red |

### Pages
| Page | Route | Features |
|------|-------|---------|
| `LoginPage.jsx` | `/login` | Email/password form, error handling |
| `RegisterPage.jsx` | `/register` | Name/email/password, 6-char minimum |
| `DashboardPage.jsx` | `/dashboard` | 4 stat cards (total, outstanding, paid, overdue) + recent invoices list |
| `InvoicesPage.jsx` | `/invoices` | Full list with filter tabs (all/pending/paid/overdue), inline delete |
| `CreateInvoicePage.jsx` | `/invoices/new` | Form with rupee amount field, date picker, description |
| `InvoiceDetailPage.jsx` | `/invoices/:id` | View + inline edit + status change + AI Reminder Generator |

### Routing
- `react-router-dom` v7 added to `client/package.json`
- `axios` added to `client/package.json`
- `client/vite.config.js` ? added dev proxy: `/api` ? `http://localhost:8080`
- All invoice routes are protected ? unauthenticated users redirected to login

## 4. Docker Compose Deployment

### Files Created
| File | Purpose |
|------|---------|
| `server/Dockerfile` | Node 20 Alpine + python3/make/g++ for bcrypt native build |
| `client/Dockerfile` | Multi-stage: Node 20 build ? nginx:alpine serve |
| `client/nginx.conf` | Serves React SPA, proxies `/api/` to `http://server:8080` |
| `docker-compose.yml` | Two services: `server` (port 8080) and `client` (port 80) |
| `.env.example` | Root env template for docker-compose |
| `server/.dockerignore` | Excludes node_modules, .env, .git |
| `client/.dockerignore` | Excludes node_modules, dist, .git |

### How to Deploy

**Step 1 ? Create your `.env` file (root of project):**
```bash
cp .env.example .env
# Edit .env and fill in:
# MONGO_URI=your_atlas_uri
# JWT_SECRET=any_long_random_string
# OPENAI_API_KEY=sk-...  (optional, for AI reminders)
```

**Step 2 ? Install new client dependencies locally (for dev only):**
```bash
cd client && npm install
```

**Step 3 ? Build and run with Docker Compose:**
```bash
docker-compose up --build
```

**Access:**
- Frontend: `http://localhost:81`
- Backend API: `http://localhost:8081`

## Environment Variables Reference

| Variable | Where | Required | Description |
|----------|-------|----------|-------------|
| `PORT` | root `.env` | Yes | Server port (default 8080) |
| `MONGO_URI` | root `.env` | Yes | MongoDB Atlas connection string |
| `JWT_SECRET` | root `.env` | Yes | Long random string for JWT signing |
| `OPENAI_API_KEY` | root `.env` | No | Enables AI reminder generation |
| `CLIENT_URL` | root `.env` | Yes | Frontend URL for CORS (e.g. http://localhost:81) |
